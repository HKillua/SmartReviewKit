## 这组文档怎么用

- 面试前快速过一遍：按顺序看四篇，能快速把“为什么这样改”和“怎么落地”串起来。
- 项目复盘时：重点看每篇里的“关键 tradeoff”和“效果与验证”。
- 后续继续改动时：保持同样模板追加新文档，不要把这组文档写成代码 diff 清单。
